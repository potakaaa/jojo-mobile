# Backlog: Decide gitignore for apps/mobile/android/

**Priority:** P2 — low urgency; cosmetic repo hygiene
**Discovered:** STAFF-001 execute session, 2026-07-13

## Problem

`apps/mobile/android/` is a generated native directory created by Expo's prebuild step. It is
currently **untracked** (shows in `git status` as `?? apps/mobile/android/`) and NOT listed in
any `.gitignore`.

This causes noise in `git status` output and risks accidentally committing the generated native
directory if someone runs `git add -A`.

## Options

1. **Gitignore it (recommended for managed Expo workflow):** Add `apps/mobile/android/` to the
   root `.gitignore` or `apps/mobile/.gitignore`. This is the standard approach when using EAS
   Build — the native directories are regenerated by EAS and don't need to be checked in.

2. **Track it (bare-workflow approach):** Commit the directory if the team wants to customize
   native Android code directly (requires ejecting from managed workflow). Not appropriate until
   EAS Build is configured and the team is ready to maintain native code.

## Recommended action

Add to `apps/mobile/.gitignore`:
```
# Expo generated native dirs (regenerated by EAS Build)
/android/
/ios/
```

Or to root `.gitignore`:
```
apps/mobile/android/
apps/mobile/ios/
```

Verify with the team whether `apps/mobile/ios/` is also untracked or already gitignored.
