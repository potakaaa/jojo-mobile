## TL;DR

Add one more check to `POST /orders`'s existing branch-validation block: after the `is_accepting_pickup` check, call `getIsOpenNow(branch.opening_hours, new Date())` and reject with a new `BRANCH_CLOSED` reason code + distinct copy if false. Give `OrderError` an optional machine-readable `reason` field so both this new check and the existing `is_accepting_pickup` check can carry distinct codes (`AC1`, `AC3`). Add the previously-missing `is_accepting_pickup` regression test (`AC4`, Locked Decision 3). **Critical finding this session: 5 existing test-branch fixtures in `orders.test.ts` seed `opening_hours: '08:00-20:00'` — a plain string, not the JSON shape `getIsOpenNow` parses — so `JSON.parse` throws and `getIsOpenNow` returns `false` for every one of them today.** Adding the new check without first fixing these fixtures would turn every existing happy-path order test red. This plan fixes all fixture sites as an in-scope, mandatory step (not a nice-to-have), pinned to an explicit "always-open" JSON opening_hours + a TZ-safe frozen `now`. No "next opening time" derivation function is built this pass — deferred as fast-follow per AC7's own conditional wording (see Decision D3 below).
