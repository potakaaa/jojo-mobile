**Filed:** 20-07-26 (STAFF-005 dashboard-home UPDATE PROCESS)
**Priority:** P2 — harness tooling bug, not a project-context bug
**Source:** discovered while reconciling `process/context/all-context.md` after STAFF-005 EXECUTE

## Problem

`node .claude/skills/vc-context-discovery/scripts/discover-context.mjs --check-routing` reports the
`GENERATED:routing` block in `process/context/all-context.md` as stale (pre-existing, unrelated to
this session's edits — reproduced against a clean file before this session's own additions).
Running `--emit-routing` to fix it instead REWROTE the block to `(no groups yet — populated during
STUDY phase, then regenerated by --emit-routing)`, wiping the real, correct `planning/` and `tests/`
group rows that were previously present and accurate. The generator apparently fails to detect the
existing `process/context/planning/all-planning.md` and `process/context/tests/all-tests.md` group
entrypoints (both have valid `context:` frontmatter per `vc-context-discovery`'s own schema) when
regenerating the table — likely a frontmatter-field mismatch between what `discover-context.mjs`
scans for and what those files actually carry.

Manually reverted the destructive rewrite this session (restored the two rows by hand) rather than
trust the generator's output.
