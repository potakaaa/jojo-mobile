# PLAN — Display Font Tone Swap + Payment Method Label Overflow Fix

Date: 20-07-26
Status: DRAFT — VALIDATE PASS (validate-contract written 20-07-26)
Complexity: SIMPLE (single-session, ~7 touchpoints, no schema/auth/API/billing surface; one dependency removed). Calibration confirmed against `process/context/planning/all-planning.md` and `process/context/all-context.md`.

**TL;DR:** Two independent, locked-scope presentation changes. Item 1 repoints the single display-bold font token to an already-loaded weight and cleans up the now-dead Fredoka package + PRD prose. Item 2 makes the payment-method label wrap instead of overflowing. Everything is behind the shared `packages/ui` token map / component — no per-screen edits. All hard proof is Fully-Automated; visual confirmation is the standing Agent-Probe RN-runner gap (Known-Gap, not a blocker).

---

## Overview / Goals

1. **Display font tone swap** — repoint `FontFamily.display.bold` from `Fredoka_700Bold` to `PlusJakartaSans_800ExtraBold` (already loaded, currently zero consumers). This uniformly changes all 65 display-bold heading call sites across ~40 files with a single token edit. Then remove both now-orphaned Fredoka weights from the `useFonts()` load list + their import, drop the `@expo-google-fonts/fredoka` dependency, and amend PRD §12 to describe the more grown-up display tone.
2. **Payment method label overflow fix** — add `numberOfLines={2}` + shrink-safe flex to the label `Text` in `payment-method-selector.tsx` so a long label ("Credit/debit card") wraps to a second line instead of overlapping the "Unavailable" badge, matching the caption's existing wrap pattern.

Non-goals: body-text tokens, `PaymentMethod` enum, `isMethodDisabled` semantics, `checkout.tsx`/`confirmation/[orderId].tsx`, `apps/admin`, any broader typography audit. (Full list: SPEC §Out Of Scope.)
